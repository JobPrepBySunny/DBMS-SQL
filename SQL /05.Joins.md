# üü¶ **Joins**

Joining means:
üëâ ‚ÄúCombine rows from two tables based on a matching column.‚Äù

Usually match happens using a **foreign key** like `users.id = orders.user_id`.

---

## **1) INNER JOIN**

**Meaning:**
‚ÄúGive me only the matching rows from both tables.‚Äù

**Explanation:**

* Shows rows where the join condition matches in **both** tables.
* If a row does not have a match, it is excluded.

**Example:**

```sql
SELECT u.name, o.amount
FROM users u
INNER JOIN orders o ON u.id = o.user_id;
```

This means:
‚û°Ô∏è Show user name and order amount **only for users who have placed an order**.

---

## **2) LEFT JOIN** (or **LEFT OUTER JOIN**)

**Meaning:**
‚ÄúGive me all rows from the LEFT table, and matching rows from the RIGHT table.‚Äù

**Explanation:**

* Left table ‚Üí never lost
* Right table ‚Üí matched values shown, unmatched become **NULL**

**Example:**

```sql
SELECT u.name, o.amount
FROM users u
LEFT JOIN orders o ON u.id = o.user_id;
```

This means:
‚û°Ô∏è Show all users, even those who never ordered.
‚û°Ô∏è For users with no orders, order amount will be NULL.

---

## **3) RIGHT JOIN** (or **RIGHT OUTER JOIN**)

**Meaning:**
‚ÄúGive me all rows from the RIGHT table, and matching rows from the LEFT table.‚Äù

**Explanation:**

* Right table ‚Üí never lost
* Left table ‚Üí unmatched values become **NULL**

**Example:**

```sql
SELECT u.name, o.amount
FROM users u
RIGHT JOIN orders o ON u.id = o.user_id;
```

This means:
‚û°Ô∏è Show all orders, even if some orders are not linked to any user.
‚û°Ô∏è Those unmatched users appear as NULL.

---

## **4) FULL JOIN** (or **FULL OUTER JOIN**)

**Meaning:**
‚ÄúGive me all rows from both tables ‚Äî matched or not.‚Äù

**Explanation:**

* Rows with matches ‚Üí normal
* Rows without matches ‚Üí NULL for missing side
* Available in PostgreSQL, not MySQL (but can be simulated)

**Example:**

```sql
SELECT u.name, o.amount
FROM users u
FULL JOIN orders o ON u.id = o.user_id;
```

This means:
‚û°Ô∏è Show every user and every order.
‚û°Ô∏è If something doesn‚Äôt match, fill the missing side with NULL.

---

## **5) CROSS JOIN**

**Meaning:**
‚ÄúCombine every row from the first table with every row from the second table.‚Äù

**Explanation:**

* Produces a **Cartesian product**
* If table A has 5 rows and table B has 3 rows ‚Üí output = 15 rows
* Used rarely, usually for generating combinations

**Example:**

```sql
SELECT * FROM colors CROSS JOIN sizes;
```

This means:
‚û°Ô∏è Pair each color with each size.

---

## **6) SELF JOIN**

**Meaning:**
‚ÄúJoin a table with itself.‚Äù

**Explanation:**

* Used when rows are related inside the same table
* Common in hierarchical or relationship data
* Requires table aliasing

**Example:**

```sql
SELECT e1.name AS employee, e2.name AS manager
FROM employees e1
JOIN employees e2 ON e1.manager_id = e2.id;
```

This means:
‚û°Ô∏è From the same table, match each employee with their manager.
